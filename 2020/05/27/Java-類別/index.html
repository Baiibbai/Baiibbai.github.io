Java-類別

類別(Class)

類別是由「資料成員」與「函數成員」封裝而成

  • 資料成員(data member、field) : 屬性(attribute)、變數(variable)
  • 函數成員(function member) : 方法(method)

Ex.
類別 : 矩形
資料成員 : 長、寬等
函數成員 : 計算面積與周長

一個子類別只能繼承一個父類別
但可以有多個介面，介面可以繼承多個介面

main()方法 : Java進入點，為一個public成員(Member)

  1. 不需要產生物件實例就要能夠被執行，所以必須是”static”
  2. void 不需回傳

存取修飾子

  • public(公開) : 任何類別跟自己都可以訪問
  • protected(保護) : 只有子類別、相同資料夾內的類別與自己可以訪問
  • default(預設) : 在相同資料夾內的類別與自己可以訪問
  • private(私有) : 只有自己
  • static(靜態)
    • JVM自動載入靜態變數與副程式至記憶體
    • 全域變數
    • 資料共用
    • 一開始就分配好記憶體，一開始就有實體，不需要new

類別的定義格式

  • 類別名稱有大小寫之分
  • 類別名稱的命名和一般變數一樣，不能和關鍵字與保留字相同

建立新的物件

  • 類別可以視為一個模板，要利用它來建立屬於該類別的物件
  • 新建立好的物件，即具有該類別的特徵，也就是每個物件均擁有其所屬類別的資料成員與函數成員
  • 由類別所建立的物件稱為instance，有些稱為「實例」

Ex. 先打造一個矩形的模板(定義類別)，再以此模板製造矩形(建立物件)

宣告與建立物件

(1) 以類別名稱宣告變數
(2) 利用new建立新的物件，並指派給先前所建立的變數

指向物件的變數

rect1是指向由CRectangle類別所建立的物件，並非物件本身

this關鍵字

刻意強調「物件本身的field」

使用多載常犯的錯誤

不能設計出引數個數和型態完全相同，而只有傳回值型態不同的多載

建構元

當程式中沒有撰寫建構元時，Java會自動呼叫預設的建構元

預設建構元三特點

  • 建構元的名稱=類別名撐
  • 建構元裡沒有引數
  • 不做任何事，也就是建構元內沒有任何的敘述

實例變數與實例函數

彼此之間不能共享

  • 實例變數 : 變數各自獨立，且存於不同的記憶體
  • 實例函數 : 必須先建立物件，再利用物件來呼叫

類別變數與類別函數

由所有的物件共享

  • 類別變數 : 在變數前之前要加上「static」修飾子，
  • 類別函數 : 內部不能使用this關鍵字、無法取用「實例變數」與「實例函數」
