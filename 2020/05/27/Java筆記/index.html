Java筆記

基本型態(Primitive type)

  • 數值型
    • 整數 : byte、short、int、long
    • 浮點 : float、double
  • 字元 : char
  • 布林 : boolean

類別型態(Class type)

  • 類別 : Class
  • 介面 : interface
  • 抽象類別 : abstract
  • 列舉 : enum

建構子

用來設定初值，且載入記憶體使用完就會釋放

  • 宣告屬性變數(成員變數、欄位)的初值
  • 宣告時，名稱必須與類別名稱相同
  • 建構子方法前不可宣告任何傳回值型別
  • 使用new才會被呼叫
  • 預設建構子 : 每個類別至少有一個建構子。若沒有宣告建構子，JAVA會提供一個預設建構子(無參數、無實作)

this()用於呼叫類別物件中同名的另一個建構子方法
this()必須寫在建構子方法的第一行

  • 類別 : 事物的一個抽象，具有共同特徵的一類事物的一個抽象
  • 物件 : 類具體的某一實例
  • 所以實例(instance)或物件(object) -> 物件實例
  • instance of 判斷記憶體位置是哪一子類別

物件導向特性

封裝(Encapsulation)

  • 目的 : 保護重要的屬性變數或副程式(讓外部程式不可以隨意存取)
  • 方法 : 將變數或副程式宣告為private(私有)
  • 開發 : 有限度的開放被保護的屬性變數或副程式
    • getter 取值 setter設值
    • 必須檢查值的合法性(有效性)和安全性

繼承(Inheritance)

  • 目的 : 不開發重複功能程式碼
  • 方法 : 使用extends
    • 子類別(subclass) extends 父類別(superclass)
  • 開發 : 子類別在使用時，會呼叫父類別的建構子
    • 如果未特別告知，則會去呼叫父類別的預設建構子
    • 如有super，則看super撰寫方法

呼叫自己類別方法 : this()

  • 只能寫在建構子的第一行
  • 呼叫自己的屬性變數 : this.屬性
  • 呼叫自己的屬性方法 : this.方法()

呼叫父類別的建構子 : super()

  • 只能寫在建構子的第一行，如果子類別想要同時呼叫自己與父親的建構子，必須先呼叫自己的建構子，再由自己的其他建構子去呼叫父類別的建構子
  • super.屬性 & super.方法()

覆寫(Override)

  • 外部程式無法存取父類別中的method
  • Class包含「成員變數(Filed)」、「建構子(constructor)」、「方法(method)」
  • Filed & Method可以使用存取修飾子，但建構子不行
  • Method 可以擁有同名但參數有所不同的「多載(Overload)」
  • 建構子內可使用this()->子類別 super()->父類別

overload

  • 有無繼承都會發生
  • 有個同名副程式，但參數列中的參數數量、型別、型別順序必須不同

override

  • 繼承才會發生
  • 父子類別中的方法宣告必須一致

抽象類別(abstract class)

未定義完全的類別，所以不能被用來生成物件，只能被擴充，擴充後完成未完成的抽象方法定義

  • 若類別中有一個方法未實作任何程式碼，則該方法必須宣告為抽象方法，此類別必須為抽象類別
  • 父類別為抽象類別，其所產生的抽象方法必須由子類別來實作
  • 如果子類別沒有全部實作出父類別提供的所有抽象方法，子類別也須宣告為抽象
  • 抽象方法的存取修飾子不可以是static,final,private
  • 方法不可以有程式碼{}
  • 類別不可以有new
  • 類別不一定要有方法，目的強迫繼承

abstract

  • 不可寫在變數
  • 使用在方法上時，方法不能實作程式碼

final

  • 如果寫在變數上時 -> 常數，數值不可改變
  • 如果寫在方法上時 -> 不可被覆寫(override)
  • 如果寫在類別上時 -> 不可被extends

抽象方法(abstract method)

規範操作方法，並在執行時期可操作各種子類別的物件實例

介面(interface)

規範公開的操作介面，一個實作介面的某類別，必須實作介面中所有已規範的操作方法，一個類別可以實作多個介面，實現多重繼承，表示一個類別可以身兼多個角色與職責

特性

  • 介面宣告的方法，沒有實作內容
  • 介面方法由類別來實作
  • 多個無關類別可以實作同一介面
  • 一個類別可以實作多介面
  • 介面定義一個JAVA型別

存取修飾子 final

  • 不能被繼承
  • 不能被覆蓋
  • 不能被修改值

物件 object

是所有類別的根

  • toString 回傳物件本身描述
  • finalize 回收沒被使用的object

多型(polymorphism)

使用父類別宣告一個參考名稱，並讓其參考至子類別的物件實例，並以父類別上的公開介面來操作子類別上對應的公開方法

  • 目的 : 使用同一變數可操作多種空間

  • 注意

    • 要有繼承關係
    • 要有覆寫(override)
    • 父類別參考變數指向 子類別物件實例

  • 一個物件實例只有一個型別。變數是多型的，因為它們可指向有繼承關係的父子型別的物件實例

  • 有繼承關係的架構下，單一的物件參考變數可被宣告為多型

  • 複寫機制讀的變化

  • non-static方法

    • 沒覆寫 : 執行父類別
    • 有覆寫 : 執行子類別已覆蓋方法

  • static

    • 使用父同名static方法

  • 屬性變數

    • 使用父同名屬性變數

列舉(enum)

  • public final static
  • 無法更改
  • ==,equals()
  • 內容值在內部排序與宣告一樣
  • java.lang.Compareable介面
  • 內容值的資料型別等於該enum型別
